# 2. 데이터베이스와 빅데이터

## 2.1 데이터 베이스

### 개념
- 수많은 데이터를 체계적으로 수집, 저장하여 다양한 용도와 방법으로 활용할 수 있게 정리된 정보의 집합체

### 정의
- EU : 법적 보호의 대상으로 체계적이고 조직적인 방법으로 정리되어 개별적으로 접근할 수 있는 데이터의 집합
- 우리나라 : 소재를 체계적으로 배열 또는 구성한 편집물로써, 개별적으로 그 소재에 접근하거나 그 소재를 검색할 수 있도록 한 것
- 한국정보통신기술협회(TTA) : 여러 사용자의 요구에 대응하여 데이터를 받아들이고 저장/공급하기 위해 논리적으로 연관된 데이터를 편리하게 구성한 집합체

### DB와 DBMS
- DB : 체계적으로 정리된 데이터의 집합
- DBMS : 이러한 DB를 효율적으로 관리(구축, 유지, 활용)할 수 있도록 도와주는 소프트웨어

### 특징
- 통합된 데이터 : 동일한 내용의 데이터가 중복없이 관리
- 저장된 데이터 : 컴퓨터가 접근할 수 있는 매체에 물리적으로 저장
- 공용 데이터 : 하나의 데이터의 여러 사용자가 공유하고 활용
- 변화하는 데이터 : 삽입, 수정, 삭제를 통해 지속적으로 갱신

## 2.2 데이터 베이스의 활용

### 기업 내부의 데이터 베이스 시스템
- 활용 목적에 따라 2가지 시스템으로 구분
    1. OLTP (Online Transaction Processing) : 실시간으로 발생하는 일상적인 거래 데이터를 신속하고 안정적으로 입력하고 처리하여 시스템의 운영 효율성을 유지하는데 중점을 둔다.
    2. OLAP (Online Analytical Processing) : OLTP를 통해 축적된 대규모 데이터를 다양한 관점에서 분석하고 조회하여 경영진의 전략적인 의사 결정을 지원하는데 중점을 둔다.

### 주요 기업 정보 시스템
- 전사적 자원 관리 (Enterprise Resource Planning, ERP)
    : 기업의 핵심 자원(재무, 인사, 생산등)을 단일 시스템으로 통합 관리하여 전사적 효율 극대화
- 공급망 관리 (Supply Chain management, SCM)
    : 원자재 공급자부터 최종 고객까지 이어지는 공급망의 모든 흐름을 최적화하여 효율을 높인다.
- 고객 관계 관리 (Customer Relationship Management, CRM)
    : 고객 데이터를 분석하여 고객과의 관계를 효과적으로 관리/마케팅에 활용
- 지식 관리 시스템 (Knowledge Management System, KMS)
    : 지식과 노하우를 공유/축적하여 문제 해결 능력 향상
- 비즈니스 인텔리전스 (Business Intelligence, BI)
    : 데이터를 수집, 분석하여 합리적인 의사결정을 지원
- 기업 애플리케이션 통합(Enterprise Application Intergration, EAI)
    : 기업 내 분리된 다양한 애플리케이션들을 서로 연결하고 통합하여 데이터 흐름을 원활하게 하는 기술
- 실시간 기업 (Real-Time Enterprise, RTE)
    : 기업 업무 정보를 실시가능로 파악하고 대응하여 신속한 의사결정이 가능한 환경

## 2.3 데이터 베이스의 종류

### 2.3.1 관계형 데이터 베이스 (Relational DataBase, RDB)
- 정의 : 행과 열로 구성된 2차원 표(Table)로 저장하는 방식
- 구조 : 명확한 구조를 가지는 정형데이터
- 특징 : 다루기 쉽고, 중복 없이 일관성을 유지하는 데 강점
- 종류 : Oracle, MySQL, MS-SQL, MariaDB

### 2.3.2 NoSQL(Not Only SQL)
- 정의 : 대용량의 비정형 데이터를 빠르고 유연하게 처리하는데 초점을 맞춘 모델
- 특징 : 유연성, 확장성, 분산 처리 및 빅데이터 분석 환경에서 뛰어난 성능 발휘
- 종류
    - Document-Oriented(문서 지향)
        : 데이터를 JSON/XML과 같은 무서형태로 저장하며 계층적 구조 표현에 용이 (ex) MongoDB, CouchDB)
    - Key-Value(키-값)
        : 하나의 키 - 하나의 값 => 빠른 조회가 필요한 환경 (ex) Redis, DynamoDB)
    - Column-Oriented(컬럼 지향)
        : 데이터를 열단위로 저장하여 대규모 데이터 분석 및 읽기 연산에 최적화되어있다. (ex) Cassandra, HBase)

### 2.3.3 기타 데이터 베이스 모델
- 계층형 DBMS
    : 데이터를 Tree구조로 저장하며 회사 조직도처럼 상하관계 표현
    검색속도 빠름, 데이터 간 연결 구조 고정적, 유연성 낮음, 구조 변경 어려움
- 네트워크형 DBMS
    : 네트워크처럼 복잡한 구조 표현, 설계/유지 보수 어려움
- 분산형 DBMS
    : 하나의 데이터베이스를 여러 물리적 위치의 서버에 분산하고 네트워크를 통해 통합적으로 관리

## 2.4 SQL

### 2.4.1 명령어 분류
- DDL(데이터 정의어)
    : 데이터의 그릇을 다루는 명령어
    - CREATE : 테이블 생성
    - ALTER : 테이블 구조 변경
    - RENAME : 이름 변경
    - DROP : 테이블 삭제
- DML(데이터 조작어)
    : 데이터를 조작하는 언어
    - SELECT : 데이터 조회
    - INSERT : 삽입
    - UPDATE : 변경
    - DELETE : 데이터 삭제
- DCL(데이터 제어어)
    : 보안과 권한
    - GRANT : 권한 부여
    - REVOKE : 권한 회수
- TCL(트랜잭션 제어어)
    : 작업 단위(트랜잭션) 제어
    - COMMIT : 트랜잭션 확정
    - ROLLBACK : 철회
    - SAVEPOINT : 중간 저장점 설정

### 2.4.2 함수 문법

#### 기본 문법
- SELECT, FROM, WHERE

#### 집계 함수
- COUNT, SUM, AVG, MAX, MIN

#### 조건 필터링
- WHERE : 그룹 전 필터링
- GROUP BY : 그룹화 기준 지정
- HAVING : 그룹화 후 필터링

## 2.5 빅데이터
- 정의 : 기존 데이터 처리 방식으로는 저장/관리/분석이 어려운 대규모의 복잡한 데이터 집합 + 분석을 위한 기술/방법/인력/조직 역량을 모두 포괄하는 개념
- 핵심 특징(3V) : Volume, Variety, Velocity
- 추가 특징(4V) : Value, Veracity, Visualization, Variability
- 등장 배경 : 데이터 양적 증가, 산업계 변화, 학계 변화, 관련 기술 발전
- 활용 기본 기법 7가지
    - 회귀, 분류, 기계학습
    - 연관 규칙 학습 (Association Rule Learning)
    - 유전 알고리즘 : 자연선택과 돌연변이의 원리를 이용해 최적의 해를 찾아가는 기법
    - 감정 분석 : 텍스트에 담긴 긍정/부정 등 주관적 감성을 추출하여 고객 반응 파악한다 ex) 후기
    - 소셜 네트워크 분석 (SNA) : 사람 간의 연결 구조를 분석하여 정보 확산의 중심이나 위험 그룹을 파악한다

## 2.6 인프라
- 목적, 데이터, 구조화, 특징, 범위

1. 데이터 웨어 하우스 - DW
    : 전사적 통합 분석, 정형 데이터, 저장 구조화, 통합성/시계열성/비휘발성, 전사적(기업 전체)
2. 데이터 레이크 - DL
    : 모든 데이터 원본 저장, 모든 유형, 분석 구조화, 원본 그대로 저장, 전사적(모든 소스)
3. 데이터 마트 - DM
    : 특정 부서의 신속한 분석, 정형 데이터(DW에서 추출), DW의 축소판(특정 주제), 저장 구조화, 특정 부서/주제
