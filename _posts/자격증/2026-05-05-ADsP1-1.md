# 1. 데이터의 이해

## 1.1 데이터의 정의
- 데이터 : 가공되지 않은 객관적인 사실 그 자체
- 정보 : 데이터를 특정 목적을 위해 가공하고 처리하여 의미를 부여한 결과물

| 데이터 특성 | 의미 |
| --- | --- |
| 존재적 특성 | 해석되기 전의 순수한 사실 자체 |
| 당위적 특성 | 가공 및 해석을 통해 |

## 1.2 데이터의 유형
### 정성적 vs 정량적
- 정성적 데이터 : 언어나 문자로 표현되어 수치화가 어려운 데이터
- 정량적 데이터 : 숫자로 표현되고 계산 가능한 데이터

### 구조에 따른 데이터 분류
- 정형 데이터 : 고정된 행과 열을 가지는 관계형 데이터베이스
- 비정형 데이터 : 정해진 구조가 없는 데이터
- 반정형 데이터 : 고정된 행과 열은 없지만 태그나 계층구조로 구분이 가능한 형태 ex) json

## 1.3 데이터 품질
### 정의
- 사용자의 목적에 맞게 정확성, 완전성, 일관성을 유지하는 정도

### 핵심 요소
- 정확성, 완정성, 일관성, 적시성, 유효성, 고유성

### 품질 저하의 주요원인
- 데이터 입력 오류
- 시스템 간 연동 오류 : 데이터 불일치 & 최신 상태 미반영
- 데이터 표준 미준수 : 규칙 통일 x
- 수집 주기 지연 : 갱신 주기를 지키지 않음
- 관리 부주의 : 잘못된 데이터가 남아있음

### 관리 방법
- 데이터 표준화
- 검증 절차 도입
- 정기 점검
- 데이터 정제 : 오류, 중복, 누락 데이터를 수정/삭제하여 신뢰성 확보
- 담당자 교육

## 1.4 DIKW 피라미드

### 단계별 정의
>>>> 지혜 : 지식이 축적되어 원리를 이해하고 왜 그래야하는지 통찰하여 미래를 예측할 수 있는 상태
>>> 지식 : 정보가 모여 패턴을 이루거나 어떻게 활용할 수 있는지 알게 된 상태
>> 정보 : 데이터를 수집하고 분류하여 맥락이 더해진 의미있는데이터가 된 단계ㄱ 
> 데이터 : 아무런 의미가 부여되지 않은 순수한 사실

## 1.5 메타 데이터의 개념과 역할

### 정의
- 데이터에 대한 데이터
    - ex) 책이 데이터라면 책의 제목, 저자, 출판일

### 주요 역할
- 쉽게 검색하고 분류할 수 있음
- 의미와 배경을 제공하여 이해와 활용 도음
- 이력 관리하여 품질 향상

### 종류
- 구조 메타 데이터 : 구조와 형식을 설명
- 기술 메타 데이터 : 문서의 제목, 작성자, 키워드 등 내용 설명
- 관리 메타 데이터 : 파일 생성일, 접근 권한 등 관리에 필요한 정보
