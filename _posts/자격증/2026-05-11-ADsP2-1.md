# 1. 데이터 분석 기획의 이해
## 1.1 분석 기획과 분석 방법론
- 정의 : 실제 분석을 시작하기 전에 분석 과제를 정의하고 의도한 결과를 얻기 위한 관리 방안을 미리 세우는 작업
- 특징 : 도메인, 기술 역량, 품질 통제 구조화
- 분석 대상/방법에 따른 4가지 분석 주제
    1. 방법 o 대상 o => 최적화
    2. 방법 o 대상 x => 통찰
    3. 방법 x 대상 o => 솔루션
    4. 방법 x 대상 x => 발견
- 목표 시점 별 분석 기획
    - 과제 중심 : 빨리 해결, 빠른시간, 문제해결
    - 마스터 플랜 : 지속 해결, 정확성, 긴시간, 문제정의
- 고려 사항
    - 가용 데이터, 활용방안, 유스케이스 탐색, 장애 요소 사전 계획(회피,전이,완화,수용)

## 1.2 분석 방법론 모델
1. 폭포수 모델(Waterfall Model)
    - 단계가 순차적으로 진행되는 구조
    - 현재 단계가 완료되어야 다음 단계 가능
    - 문제 발생시 이전 단계 피드백
2. 프로토타입 모델(Prototype Model)
    - 사용자 요구사항이 불명활할 때 부분적인 시제품 만들고 점진적으로 완성도를 높여가는 모델
3. 나선형 모델(Spiral Mode)
    - 반복을 통해 점증적으로 개발 
    - 프로토랑 달리 위험 제거에 초점을 맞춘 모델
4. 계층적 프로세스 모델(Hierarchical Process Model)
    - 단계 -> 태스트 -> 스텝의 3계층 구조
    - 스텝은 입력 - 처리/도구 - 출력으로 구성 => 품질 통제와 책임 추적

## 1.3 전통적인 분석 방법론
1. KDD(Knowledge Discovery in Database)
- 정의 : 원천 데이터에서 유의미한 패턴과 지식을 찾아내는 절차적 접근
- 흐름 : 데이터 선택, 전처리, 변환, 데이터 마이닝, 해석/평가
- 특징 : 빠른 실험에 유리, 업무이해/운영 단계가 명시적 분리 x
2. CRISP-DM(Cross Industry Standard Process for Data Mining)
- 정의 : 재사용 가능하도록 정리된 6단계/4계층의 표준 방법론
- 프로세스 6단계 : 업무 이해, 데이터 이해, 데이터 준비, 모델링, 평가, 전개
- 특징 : 업무-데이터-모델-운영을 하나로 묶음, 품질/일정 관리에 용이
